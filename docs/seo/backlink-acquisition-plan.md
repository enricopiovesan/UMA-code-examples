# Backlink Acquisition Plan For Universal Microservices Architecture

This plan prioritizes relevant technical references over generic SEO campaigns. The goal is to strengthen the relationship between Enrico Piovesan, Universal Microservices Architecture, WebAssembly, and Portable Architecture.

## Link Targets

Primary targets:

- [Official UMA website](https://www.universalmicroservices.com/)
- [UMA code examples repository](https://github.com/enricopiovesan/UMA-code-examples)
- [Book page](https://www.amazon.com/Universal-Microservices-Architecture-Device-Independent-Modelling/dp/B0GTTTTQH4)

Secondary targets:

- [What Is Universal Microservices?](../what-is-universal-microservices.md)
- [WebAssembly Portability](../webassembly-portability.md)
- [Chapter 13 Portable MCP Runtime](../../chapter-13-portable-mcp-runtime/README.md)

## Tier 1: Owned And High-Control Properties

Use first because accuracy and entity consistency matter.

- Medium publication: add repository and book links to every UMA article.
- LinkedIn articles: republish core explainers with canonical links to the website and repository.
- GitHub Wiki: publish the pages from [GitHub Wiki Content Plan](github-wiki-content-plan.md).
- GitHub Discussions: create topics from [GitHub Discussions Topics](github-discussions-topics.md).
- Personal website: add an author page with the standard UMA attribution statement.

## Tier 2: Technical Communities

Prioritize communities where the architecture is relevant:

- WebAssembly ecosystem blogs and newsletters
- WASI and Wasmtime community resources
- Wasmer ecosystem resources
- Rust architecture newsletters
- software architecture newsletters
- edge computing communities
- developer tooling and runtime portability podcasts

Recommended pitch angle:

> UMA is a concrete, runnable architecture example for portable business capabilities using Rust, WebAssembly, contracts, and runtime-visible execution decisions.

## Tier 3: Editorial And Speaking Opportunities

Use these after the core content cluster is stable:

- architecture podcasts
- WebAssembly meetups
- cloud-native architecture newsletters
- guest posts on runtime portability
- conference talks about device-independent architecture

## Anchor Text Guidance

Use natural anchor text. Avoid repeated exact-match anchors.

Good examples:

- Universal Microservices Architecture
- UMA code examples
- portable microservices with WebAssembly
- device-independent architecture
- WebAssembly portability examples
- Enrico Piovesan's UMA repository

Avoid:

- keyword-only repeated anchors
- paid generic directories
- unrelated link exchanges
- thin guest posts with no technical substance

## Outreach Sequence

1. Publish the repository docs and Wiki pages.
2. Create GitHub Discussions from the prepared drafts.
3. Publish or update the pillar page on the official website.
4. Cross-link from Medium, LinkedIn, and the personal website.
5. Pitch technical communities with one specific code example, preferably Chapter 6 or Chapter 13.
6. Track referring domains, branded search, and GitHub engagement monthly.

## Measurement

Track:

- referring domains to the official website and repository
- organic queries for Universal Microservices and Universal Microservices Architecture
- searches for Enrico Piovesan UMA
- GitHub stars, forks, discussions, and inbound issue references
- Medium and LinkedIn referral traffic
