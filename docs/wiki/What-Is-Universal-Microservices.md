# What Is Universal Microservices?

Universal Microservices is the practice of designing software capabilities so their core behavior can execute across more than one runtime while preserving the same contract and governance model.

A universal microservice is not just a small backend service. It is a portable capability with a stable boundary, deterministic behavior, and explicit runtime integration.

## Why It Matters

Distributed systems often duplicate the same behavior across:

- browser code
- backend services
- edge functions
- workflow engines
- AI-assisted execution paths

That duplication creates drift. UMA reduces drift by keeping the core behavior portable and keeping runtime decisions visible.
